Packet Tracer - Monitor Your Network Using a Network Controller

Objectives

A network controller allows you to manage, monitor, and configure supported network devices via a GUI and APIs.

Note: API usage is beyond this activity.

You will set up a network controller and monitor a configured network.

Part 1: Implement a Network Controller

Part 2: Monitor the Network

Required Resources

Latest Packet Tracer version

Part 1: Implement a Network Controller

Step 1: Connect a Network Controller to Office-SW1.

a. In the Wiring Closet move the preconfigured Network Controller from the Shelf to the Rack and power it on.

b. Inspect Rear; connect Network Controller GigabitEthernet0 to Office-SW1 GigabitEthernet1/0/19 (Copper Straight-Through).

The controller IP is 192.168.20.5.

Step 2: Verify connectivity.

a. Exit Closet (Back level / Alt+Left).

b. From Office-Admin open Desktop > Command Prompt.

1) Run ipconfig (expect 192.168.20.x /25).

C:\> ipconfig
FastEthernet0 Connection IPv4 Address....................: 192.168.20.10 Subnet Mask.....................: 255.255.255.128 Default Gateway.................: 192.168.20.1

Note: If no address, ensure DHCP selected.

2) Ping controller (192.168.20.5) ‚Äî should succeed.

C:\> ping 192.168.20.5
Reply from 192.168.20.5: bytes=32 time<1ms TTL=128 (x4)

If failed, verify cabling and power.

c. Close Command Prompt.

Part 2: Monitor the Network

Step 1: Access the Network Controller GUI.

a. Office-Admin > Web Browser > browse to 192.168.20.5.

b. Login: admin / Cisco123.

c. Review Dashboard (refresh updates status).

Question:

What menu items are listed?

Answer Area

Dashboard, Provisioning, Assurance, Policy, API Docs.

Step 2: Document the network.

Use Notes in Packet Tracer to record discovered device names and IPs.

a. Controller: Dashboard Menu > Provisioning > NETWORK DEVICE.

b. Place a note listing each device (e.g. Office-SW1: 192.168.20.4).

Step 3: Discover new wireless hosts.

Connect wireless endpoints and rediscover.

a. Reopen controller if needed.

b. Dashboard Menu > Assurance > HOSTS (observe list).

c. Enable Wireless0 on Office-Tablet and Smartphone; wait for DHCP (192.168.2.x /24). Use Fast Forward Time if required.

d. Provisioning > DISCOVERY > select Office LAN > START.

e. Return to HOSTS; verify new hosts appear.

Question:

What IPs were assigned to Office-Tablet and Smartphone?

Answer Area

Any addresses in 192.168.2.11 ‚Äì 192.168.2.254 (unique per host).
