Packet Tracer - Manage and Configure your Network using a Network Controller

Objectives

In this activity, you will set up a network controller to monitor a configured network via the GUI.

Part 1: Deploy a Network Controller

Part 2: Device Discovery

Part 3: Add a New Network Device to the Office Network

Required Resources

Latest Packet Tracer version

Part 1: Deploy a Network Controller

Step 1: Connect a Network Controller to Office-SW1.

a. In Physical Mode, navigate to the Wiring Closet (Equipment Cabinet).

b. Move the Network Controller from the Shelf to the Rack.

c. Connect Network Controller GigabitEthernet0 to Office-SW GigabitEthernet1/0/19 (Copper Straight-Through).

d. Open the Network Controller.

e. Power it on (Physical tab).

f. Global Settings: default gateway 192.168.20.1; DNS 192.168.20.126.

g. GigabitEthernet0: Static 192.168.20.5 / 255.255.255.128.

Step 2: Verify connectivity.

a. From Laptop (in Closet) ping 192.168.20.5.

b. Laptop > Desktop > Command Prompt.

c. Run ping (should succeed).

C:\> ping 192.168.20.5
Reply from 192.168.20.5: bytes=32 time<1ms TTL=128 (x4)

If unsuccessful, recheck addressing.

Part 2: Device Discovery

Configure discovery so the controller finds Office devices.

Step 1: Access the controller GUI.

a. Laptop > Web Browser > browse to 192.168.20.5.

b. Register user: admin / Cisco123 (Setup).

c. Log in with created credentials.

d. View Dashboard (maximize window).

Step 2: Create the credentials list.

a. Dashboard > Provisioning > CREDENTIALS.

b. + CREDENTIAL.

c. Username student, Password StudentPass, Enable Password Cisco123, Description Student Profile (OKAY).

Step 3: Create a Discovery Process.

a. Dashboard Menu > Provisioning > DISCOVERY.

b. + DISCOVERY.

c. Type: CDP; Name: Office - LAN; IP: 0.0.0.0; CLI Credential List: student - Student Profile (ADD).

d. Wait for completion.

e. Open Office - LAN to view discovered devices (hover to correlate rack items).

Question

Why do some devices have multiple entries?

Answer Area

Devices with multiple configured IPv4 interfaces create one discovery entry per IPv4 address.

f. Return to Dashboard to review overall status.

g. Close browser (leave Laptop window open).

Part 3: Add a New Network Device to the Office Network

Add a new switch and observe discovery updates.

Step 1: Deploy a new switch.

a. Connect Office-SW2 GigabitEthernet0/1 to Office-SW GigabitEthernet1/0/5.

b. Laptop > Terminal (OK) to access Office-SW2 CLI.

c. Paste configuration:

enable configure terminal hostname Office-SW2 interface vlan 20 ip address 192.168.20.7 255.255.255.128 no shut enable secret Cisco123 username student privilege 1 password StudentPass line vty 0 4 login local interface range g0/1-2 switchport mode trunk switchport trunk native vlan 20 interface range f0/1-24 switchport mode access switchport access vlan 2 vlan 2 name UserNetwork vlan 20 name Management end copy run start

Step 2: Review via Network Controller.

a. Log into controller GUI.

b. Dashboard Menu > Provisioning > DISCOVERY.

c. View Office - LAN list (Office-SW2 not yet present).

d. Start discovery manually (START); wait.

e. Confirm Office-SW2 appears (or check Dashboard).
