Packet Tracer - Explore Device Configuration Using the CLI (console)

Objectives

In this activity, you will use terminal emulation software to connect to the console of a device and update the configuration.

Part 1: Connect to the Device Using a Console Connection

Part 2: Copy Configuration Information to the Device

Part 3: Save the Updated Configuration to the Device

Background / Scenario

A new switch (Office-SW2) is being added. Initial access is through the console because no network configuration exists yet.

Required Resources

Latest Packet Tracer version

Instructions

Part 1: Console Connection

Step 1: Open a console session.

a. Click Laptop0 (Physical workspace).

b. Desktop > Terminal > OK.

c. Wait for "Press RETURN to get started!" and press ENTER.

Question:

What prompt appears?

Answer Area

Switch>

Step 2: Enter Privileged EXEC.

a. At Switch> type ? (lists User EXEC commands).

b. Type enable.

Switch> enable
Switch#

Prompt changed (now Privileged EXEC).

c. Type ? again (page with spacebar).

Question:

Last command in Privileged EXEC list?

Answer Area

write

d. show running-config (CTRL+C to stop if needed).

Part 2: Apply Configuration

Step 1: Enter global config.

Switch# configure terminal
Switch(config)#

Step 2: Paste commands.

hostname Office-SW2
line con 0
password Cisco123
login
end

Prompt shows new hostname.

Step 3: Test console password.

a. exit to login prompt.

b. Press ENTER.

Question:

What message appears?

Answer Area

User Access Verification, Password:

c. Enter Cisco123.

Part 3: Save Configuration

Office-SW2# copy running-config startup-config
Destination filename [startup-config]? (ENTER)
Building configuration... [OK]

a. reload (confirm).

b. After reboot, verify password prompt.

End of document
